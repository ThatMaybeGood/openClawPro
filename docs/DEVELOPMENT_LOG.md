## 🔧 技术决策记录

### 为什么选择 LocalDB 而不是真实后端？

**考虑因素:**
1. Demo 版本快速验证可行性
2. 避免复杂的服务器配置
3. 演示时更直观，无网络依赖
4. 可以立即运行测试

**最终选择:** 使用 LocalStorage 模拟数据库

**原因:**
- 零配置即可运行
- 跨页面数据共享
- 支持完整的 CRUD 操作
- 易于调试和测试

**未来计划:** 接入 Node.js/Python 真实后端 API

## 🐛 遇到的问题与解决方案

### Problem 1: PowerShell 脚本编码错误
**现象:** 特殊字符导致命令解析失败  
**原因:** PowerShell 对中文和特殊字符的转义问题  
**解决:** 放弃 PowerShell 脚本，改用 write 工具逐个创建文件  

### Problem 2: Git 命令执行失败
**现象:** `&&` 在 PowerShell 中不是有效操作符  
**原因:** 不同 Shell 的命令分隔符不同  
**解决:** 分步执行，每次只执行一条命令  

### Problem 3: 中文注释导致 Commit 失败
**现象:** 提交信息中文乱码  
**原因:** PowerShell 编码问题  
**解决:** Commit 信息使用英文
